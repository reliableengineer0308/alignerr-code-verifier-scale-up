Problem: String Decoding

## Problem Description
Given an encoded string, return its decoded string.

The encoding rule is: k[encoded_string]
This means the encoded_string inside the square brackets is repeated exactly k times.

You may assume:
1. The input string is always valid
2. There are no extra white spaces
3. All brackets are well-formed

## Input Format
- First line: T (number of test cases)
- For each test case: encoded string

## Output Format
- For each test case: decoded string

## Constraints
- 1 ≤ input length ≤ 30
- k is a positive integer
- input consists of lowercase English letters, digits, and square brackets

## Example 1: Nested Encoding
### Input
1
3[a2[c]]

### Output
accaccacc

### Explanation
Decoding process:
1. 3[a2[c]] → 3[acc] → accaccacc

## Example 2: Multiple Encodings
### Input
1
2[abc]3[cd]ef

### Output
abcabccdcdcdef

### Explanation
2[abc] → abcabc
3[cd] → cdcdcd
ef → ef
Combined: abcabccdcdcdef

## Example 3: Simple Case
### Input
1
3[a]2[bc]

### Output
aaabcbc

### Explanation
3[a] → aaa
2[bc] → bcbc
Combined: aaabcbc

## Algorithm Requirements
- Use stack to handle nested brackets
- Process digits and strings separately
- Time complexity: O(N) where N is output length
