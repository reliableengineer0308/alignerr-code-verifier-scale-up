Problem: Divide Two Integers

## Problem Description
Given two integers dividend and divisor, divide them without using multiplication, division, and mod operator.
Return the quotient after truncating toward zero.

This problem requires implementing integer division with the following constraints:
1. Cannot use *, /, or % operators
2. Must handle 32-bit integer overflow cases
3. Should be efficient (O(log n) time complexity)

## Detailed Explanation
The challenge is to perform integer division while:
- Handling both positive and negative numbers
- Correctly truncating toward zero (not floor division)
- Dealing with the 32-bit integer range [-2^31, 2^31-1]
- Using bit manipulation for efficiency

The key insight is that division can be implemented using subtraction and bit shifts. 
For example: 100 ÷ 3 can be computed as:
- 3 × 32 = 96 (since 2^5 = 32)
- 100 - 96 = 4
- 4 ÷ 3 = 1
- Total quotient = 32 + 1 = 33

## Input Format
- First line: T (number of test cases, 1 ≤ T ≤ 1000)
- For each test case: dividend and divisor (space-separated)
- Both dividend and divisor are 32-bit integers

## Output Format
- For each test case: quotient (32-bit integer)
- If the quotient would overflow 32-bit range, return 2^31 - 1

## Constraints
- -2^31 ≤ dividend, divisor ≤ 2^31 - 1
- divisor ≠ 0
- At least 50% of test cases will involve edge cases and large numbers

### Explanation
**Test Case 1:** 10 ÷ 3 = 3.333... → truncate to 3
**Test Case 2:** 15 ÷ 5 = 3

## Algorithm Requirements
- Use bit manipulation for O(log n) efficiency
- Handle all edge cases including overflow
- Implement without *, /, or % operators
- Return appropriate values for division by zero
- Time complexity: O(log n) where n is the absolute value of dividend
