Problem: Trapping Rain Water

## Problem Description
Given n non-negative integers representing an elevation map where the width of each bar is 1, 
compute how much water it can trap after raining.

The problem can be visualized as a series of bars of different heights. When rain falls, water 
gets trapped between the bars. The amount of water trapped at any position is determined by 
the minimum of the maximum heights to the left and right, minus the height of the current bar.

## Input Format
- First line: T (number of test cases)
- For each test case:
  - First line: N (size of the elevation array)
  - Second line: height array (space-separated non-negative integers)

## Output Format
- For each test case: total trapped water (print each result on a separate line)

## Constraints
- 1 ≤ T ≤ 100
- 1 ≤ N ≤ 2 * 10^4
- 0 ≤ height[i] ≤ 10^5

## Mathematical Formulation
For each position i, the trapped water is:
water[i] = min(left_max[i], right_max[i]) - height[i]
where:
- left_max[i] = maximum height from index 0 to i
- right_max[i] = maximum height from index i to n-1

Total trapped water = sum(water[i]) for all i where min(left_max[i], right_max[i]) > height[i]

## Example 1: Basic Case
### Input

12

### Output

## Algorithm Requirements
- Use dynamic programming with left and right max arrays
- Time complexity: O(N)
- Space complexity: O(N)

## Solution Approach
1. Precompute left_max array: left_max[i] = max(height[0..i])
2. Precompute right_max array: right_max[i] = max(height[i..n-1])
3. For each position, calculate trapped water using the formula
4. Sum all positive water values
